# astrbot_plugin_daily_note

一个 [AstrBot](https://github.com/AstrBotDevs/AstrBot) 插件，自动将聊天记录总结为日记并存入本地 SQLite 数据库。

## 功能

- **纯增量生成** — 每篇日记精确记录"消费到了第几条消息"，下次只处理新消息，日记之间零重叠
- **定时定点生成** — 按设定的天数间隔 + 指定时间点（如每天 22:00）自动生成日记，重启不丢间隔
- **日记跟随对话** — 每个对话独立生成和管理日记，删除对话后孤儿日记会自动清理
- **智能跳过** — 无新消息或新消息不足阈值时自动跳过，不浪费 token
- **手动触发** — 随时手动生成，支持 `force` 模式强制从头重新生成
- **日记管理** — 支持列表查看、按编号查看全文、删除单篇、清空全部
- **继承人格风格** — 生成日记时读取当前会话的人格设定，保持语气一致
- **完全独立** — 插件的所有交互不会进入聊天记录和 LLM prompt，不影响正常对话

## 环境要求

- AstrBot >= 4.5.7
- 至少配置一个 LLM 模型提供商

## 安装

### 方式一：通过 GitHub 仓库安装（推荐）

1. 打开 AstrBot 管理面板
2. 进入 **插件管理**
3. 点击 **安装插件**，输入本仓库地址：

```
https://github.com/976230600/astrbot_plugin_daily_note
```

### 方式二：手动安装

将本仓库克隆到 AstrBot 的插件目录：

```bash
cd /path/to/AstrBot/data/plugins
git clone https://github.com/976230600/astrbot_plugin_daily_note.git
```

然后在 AstrBot 管理面板的插件管理页面点击 **重载插件**。

## 配置

安装后在 AstrBot 管理面板 → 插件管理 → **每日日记** 中配置：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 模型提供商 | 下拉选择 | 空（使用当前会话默认模型） | 选择用于生成日记的 LLM 模型 |
| 间隔天数 | 整数 | 1 | 每隔多少天自动总结一次，最小 1 天 |
| 生成时间 | 整数 | 22 | 每天几点触发自动生成（0-23），默认晚上 10 点 |
| 最少新消息数 | 整数 | 3 | 自上次生成以来新消息不足此值则跳过，设为 1 表示有新消息就生成 |
| 自定义提示词 | 文本 | 空 | 可选，用于微调日记的写作风格 |

> 修改配置后需在插件管理页面点击 **重载插件** 使调度参数立即生效。

## 指令

所有指令仅限**管理员**使用。

| 指令 | 说明 | 示例 |
|------|------|------|
| `/daily_note` | 查看当前对话的日记列表 | `/daily_note` |
| `/daily_note <编号>` | 查看指定编号的日记全文 | `/daily_note 1` |
| `/daily_note_gen` | 手动为当前对话生成一篇日记（仅处理新消息） | `/daily_note_gen` |
| `/daily_note_gen force` | 忽略增量检查，基于全部历史强制重新生成 | `/daily_note_gen force` |
| `/daily_note_del <编号>` | 删除指定编号的日记 | `/daily_note_del 1` |
| `/daily_note_clear` | 清空当前对话的所有日记 | `/daily_note_clear` |

## 生成逻辑说明

### 增量消费模型

插件采用**纯增量消费**模式：每次生成日记时记录"消费到了聊天历史的第几条消息"，下次生成直接从该位置之后取全部新消息送入 LLM，不设条数上限。每篇日记覆盖一段独立的消息区间，不会重叠。

### 自动生成

- 插件在指定时间点（默认 22:00）检查所有对话
- 每个对话独立判断：新消息数不足 `min_messages` 阈值则跳过
- 只有产生了足够新对话内容的会话才会消耗 LLM token
- 调度间隔持久化到数据库，AstrBot 重启后不丢失

### 手动生成

- `/daily_note_gen` 使用与自动生成相同的增量逻辑，没有新消息时会提示而非重复生成
- `/daily_note_gen force` 忽略增量检查，基于全部聊天历史强制重新生成一篇日记

### 日记与对话的关系

- 日记按对话隔离，切换到不同对话时只能看到该对话的日记
- 从 WebUI 删除对话后，关联的日记会在下次定时任务时自动清理

## 数据存储

日记数据保存在 AstrBot 的 `data/astrbot_plugin_daily_note/daily_note.db`（SQLite 数据库），位于插件专属数据目录中，插件更新或重装不会丢失数据。
