# astrbot_plugin_daily_note

一个 [AstrBot](https://github.com/AstrBotDevs/AstrBot) 插件，自动将聊天记录总结为日记并存入本地 SQLite 数据库。

## 功能

- **自动定时总结** — 按设定的天数间隔，自动为所有会话生成日记
- **手动触发生成** — 通过指令立即为当前会话生成一篇日记
- **日记列表查询** — 按时间倒序列出所有日记，编号动态更新（1 = 最新）
- **按编号查看全文** — 通过编号快速获取任意一篇日记的完整内容
- **继承人格风格** — 生成日记时读取当前会话的人格设定，保持语气一致
- **完全独立** — 插件的所有交互不会进入聊天记录和 LLM prompt，不影响正常对话

## 环境要求

- AstrBot >= 4.5.7
- 至少配置一个 LLM 模型提供商

## 安装

### 方式一：通过 GitHub 仓库安装（推荐）

1. 打开 AstrBot 管理面板
2. 进入 **插件管理**
3. 点击 **安装插件**，输入本仓库地址：

```
https://github.com/your_name/astrbot_plugin_daily_note
```

### 方式二：手动安装

将本仓库克隆到 AstrBot 的插件目录：

```bash
cd /path/to/AstrBot/data/plugins
git clone https://github.com/your_name/astrbot_plugin_daily_note.git
```

然后在 AstrBot 管理面板的插件管理页面点击 **重载插件**。

## 配置

安装后在 AstrBot 管理面板 → 插件管理 → **每日日记** 中配置：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 模型提供商 | 下拉选择 | 空（使用当前会话默认模型） | 选择用于生成日记的 LLM 模型 |
| 间隔天数 | 整数 | 1 | 每隔多少天自动总结一次，最小 1 天 |
| 聊天记录条数 | 整数 | 50 | 每次送入模型总结的最近聊天记录条数上限 |
| 自定义提示词 | 文本 | 空 | 可选，用于微调日记的写作风格 |

## 指令

所有指令仅限**管理员**使用。

| 指令 | 说明 | 示例 |
|------|------|------|
| `/daily_note` | 查看日记列表 | `/daily_note` |
| `/daily_note <编号>` | 查看指定编号的日记全文 | `/daily_note 1` |
| `/daily_note_gen` | 手动为当前会话生成一篇日记 | `/daily_note_gen` |

### 使用示例

```
用户: /daily_note_gen
Bot:  正在生成日记，请稍候...
Bot:  日记生成成功：和朋友聊了新电影
      使用 /daily_note 1 查看内容

用户: /daily_note
Bot:  日记列表：

        1. [2026-03-03 23:30] 和朋友聊了新电影
        2. [2026-03-02 22:15] 讨论周末出行计划
        3. [2026-03-01 21:00] 分享了一首好听的歌

      共 3 篇日记
      使用 /daily_note <编号> 查看完整内容

用户: /daily_note 1
Bot:  [ 和朋友聊了新电影 ]
      2026-03-03 23:30

      今天和朋友聊了很久关于最近上映的那部科幻电影……
```

## 数据存储

日记数据保存在 AstrBot 的 `data/daily_note.db`（SQLite 数据库），不在插件自身目录内，插件更新或重装不会丢失数据。
